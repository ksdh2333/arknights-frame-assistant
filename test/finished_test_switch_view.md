## 集成测试

### 流程：视角切换按键保存与应用

> 涉及模块：`config.ahk`、`hotkey_control.ahk`、`key_bind.ahk`、`gui.ahk`、`saver.ahk`、`hotkey_actions.ahk`

- [x] **前置**：启动明日方舟并进入关卡
- [x] **操作**：打开 AFA 设置，切换到"快捷操作"，为"视角切换"绑定一个按键（如 `n`），点击"应用设置"
- [x] **预期**：提示"设置已应用！"
- [x] **操作**：在游戏中按下绑定的按键
- [x] **预期**：执行视角切换操作（暂停按钮左右各点一次 + 鼠标位置点两次），不抢夺鼠标焦点

### 流程：视角切换按键保存并重启后持久化

> 涉及模块：`config.ahk`、`saver.ahk`、`loader.ahk`

- [x] **前置**：已为"视角切换"绑定按键 `n` 并保存
- [x] **操作**：退出 AFA，重新启动 AFA，打开设置 → 快捷操作
- [x] **预期**："视角切换"输入框仍显示之前绑定的按键 `n`

### 流程：重置按键恢复默认值

> 涉及模块：`config.ahk`、`settings/actions.ahk`

- [x] **前置**：已为"视角切换"绑定了一个按键
- [x] **操作**：打开设置，点击"重置按键"，确认重置
- [x] **预期**："视角切换"输入框恢复为空

##### 异常路径

- [x] **异常**：为"视角切换"绑定一个已被其他快捷操作使用的按键（如 `z`，即"左键点击"的默认键）→ **预期**：保存时提示按键冲突，拒绝保存
- [x] **异常**：鼠标不在游戏窗口内时按下视角切换键 → **预期**：无操作，不触发点击
