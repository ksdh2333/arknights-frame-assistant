## 集成测试

### 流程：旧版配置自动迁移

> 涉及模块：`config.ahk`（MigrateFrameRate）、`settings/loader.ahk`

- [ ] **前置**：手动编辑 `%AppData%\ArknightsFrameAssistant\PC\Settings.ini`，将 `[Main]` 下的 `Frame` 值改为旧版序号（如 `Frame=3`，代表 90fps）
- [ ] **操作**：启动 AFA
- [ ] **预期**：打开设置窗口，"游戏内帧率"下拉框显示"90"（而非"3"）

- [ ] **前置**：再次手动检查 `Settings.ini`
- [ ] **预期**：`[Main]` 下的 `Frame` 值已被改写为 `90`（文本值）

- [ ] **前置**：确保 INI 中 Frame 值为文本格式（如 `Frame=90`）
- [ ] **操作**：重新启动 AFA
- [ ] **预期**：不报错，帧率设置正确显示"90"（迁移幂等，不会重复迁移）

### 流程：帧率影响过帧延迟

> 涉及模块：`config.ahk`（UpdateDelay）、`hotkey_actions.ahk`

- [ ] **前置**：游戏已启动，AFA 已激活
- [ ] **操作**：分别选择 30fps 和 180fps，在游戏中执行过帧操作（前进33ms/166ms）
- [ ] **预期**：不同帧率下过帧操作的延迟表现符合预期（30fps 下较慢，180fps 下较快）

##### 异常路径

- [ ] **异常**：INIi 文件中不存在 Frame 键 → **预期**：自动使用默认值 "90"，设置窗口显示"90"

- [ ] **异常**：INIi 文件中 Frame 值为无效文本（如 "abc"）→ **预期**：`UpdateDelay()` 不匹配任何分支，CurrentDelay 不改变，应用不崩溃
