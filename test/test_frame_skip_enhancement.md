## 集成测试

### 流程：三档过帧操作

> 涉及模块：`hotkey_actions.ahk`、`hotkey_control.ahk`、`config.ahk`

- [ ] **前置**：游戏已启动，AFA 已激活。为"前进16ms"分配一个按键（如 U），保存设置
- [ ] **操作**：在游戏中进入关卡，按 ESC 暂停，按下分配的按键
- [ ] **预期**：游戏画面极短时间解除暂停（约 16ms，几乎不可见延迟），然后恢复暂停状态

- [ ] **操作**：按 R（前进33ms），按 T（前进166ms）
- [ ] **预期**：33ms 和 166ms 过帧功能正常，延迟与之前行为一致

### 流程：自定义延迟生效

> 涉及模块：`gui.ahk`、`config.ahk`、`hotkey_actions.ahk`

- [ ] **前置**：将过帧档位1 改为 100ms，应用设置。游戏已启动
- [ ] **操作**：在游戏中执行"前进16ms"操作（实际延迟为 100ms）
- [ ] **预期**：过帧延迟明显变长（约 100ms），可以明显看到暂停解除的时间

- [ ] **前置**：将过帧档位3 改为 500ms，应用设置
- [ ] **操作**：在游戏中执行"前进166ms"操作（实际延迟为 500ms）
- [ ] **预期**：过帧延迟明显变长（约半秒），前进的帧数显著增多

##### 异常路径

- [ ] **异常**：过帧档位输入 0 → **预期**：USleep 直接返回（`delay_ms <= 0` → return），表现为几乎无延迟过帧

- [ ] **异常**：过帧档位输入负数 → **预期**：USleep 直接返回，不崩溃

- [ ] **异常**：过帧档位输入非数字（如字母） → **预期**：由于 Edit 控件设置了 `Number` 选项，无法输入非数字字符

### 流程：按键冲突检测

> 涉及模块：`settings/saver.ahk`

- [ ] **前置**：将"前进16ms"的按键设置为 R（与"前进33ms"相同）
- [ ] **操作**：点击"保存并关闭"
- [ ] **预期**：弹出冲突提示，阻止保存

### 流程：取消修改恢复

- [ ] **前置**：修改所有三个过帧档位的值为自定义值，观察"常规作战"标签变化
- [ ] **操作**：点击"取消"（不保存）
- [ ] **预期**：重新打开设置窗口，过帧档位恢复原值，"常规作战"标签也恢复原值
