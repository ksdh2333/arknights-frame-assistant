## 集成测试

### 流程：旧版配置自动迁移

> 涉及模块：`config.ahk`（MigrateFrameRate）、`settings/loader.ahk`

- [x] **前置**：手动编辑 Settings.ini，设置 `Frame=3`（旧版序号），**删除 Frame155 键**
- [x] **操作**：启动 AFA
- [x] **预期**：下拉框显示"90"（迁移后从 Frame155 读取文本值）

- [x] **前置**：检查 INI
- [x] **预期**：`Frame155=90`（新增），`Frame=3`（保留原值供旧版本使用）

- [x] **前置**：INI 已有 Frame155=90
- [x] **操作**：再次重启 AFA
- [x] **预期**：下拉框仍显示"90"（Frame155 已存在，迁移跳过）

### 流程：新旧版本兼容

> 涉及模块：`config.ahk`（SaveToIni、GetImportant、MigrateFrameRate）

- [x] **前置**：新版本保存 180fps → INI 有 Frame155=180, Frame=6
- [x] **操作**：回滚到旧版 AFA，检查设置
- [x] **预期**：旧版读取 Frame=6 → 显示 165fps（旧版无 180，映射为 165）

- [ ] **前置**：旧版本修改为 90fps 并保存 → Frame155 被旧版清除（已知限制）
- [ ] **操作**：升级到新版本，启动 AFA
- [ ] **预期**：MigrateFrameRate 从 Frame=3 恢复 Frame155=90（无法保留降级前的精确值）

- [x] **前置**：新版本保存 180fps → INI 有 Frame155=180, Frame=6
- [x] **操作**：回滚到旧版 AFA，检查设置
- [x] **预期**：旧版读取 Frame=6 → 显示 165fps

##### 异常路径

- [x] **异常**：INI 中无 Frame 也无 Frame155 → **预期**：使用默认值"90"
- [x] **异常**：INI 中 Frame155 值为无效文本 → **预期**：回退读旧 Frame 索引并转换，应用不崩溃

### 流程：帧率影响过帧延迟

> 涉及模块：`config.ahk`（UpdateDelay）、`hotkey_actions.ahk`

- [x] **前置**：游戏已启动，AFA 已激活
