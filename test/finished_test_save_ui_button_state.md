### 流程B：修改 → 保存并关闭 → 重开后按钮禁用

> 涉及模块：`gui.ahk`、`actions.ahk`

- [x] **前置**：设置窗口已打开
- [x] **操作**：修改任意设置项，点击"保存并关闭"
- [x] **预期**：弹出"设置已保存"提示，窗口关闭。重新打开设置窗口，两个按钮呈灰色禁用状态

### 流程C：修改 → 取消 → 重开后值未变

> 涉及模块：`gui.ahk`、`actions.ahk`

- [x] **操作**：修改任意设置项，点击"取消"

### 流程D：修改 → 点击 X 关闭 → 同取消行为

> 涉及模块：`gui.ahk`

- [x] **前置**：设置窗口已打开，已做修改
- [x] **操作**：点击"其他设置"标签（或从其他标签切走）
- [x] **预期**：弹出"修改尚未保存，确定离开此页面吗？"

- [x] **操作**：同上，但在确认弹窗中点击"否"
- [x] **预期**：停留在当前标签页，按钮保持可点击状态，修改未被丢弃

### 流程G：重置按键后按钮禁用

> 涉及模块：`gui.ahk`、`actions.ahk`

- [x] **前置**：已进入"常规作战"标签页

##### 异常路径

- [x] **异常**：AFA 运行中游戏进程退出（如关闭明日方舟）→ **预期**：不影响设置窗口按钮状态逻辑
