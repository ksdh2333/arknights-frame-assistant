## 集成测试

### 流程：启用系统事件自动启动

> 涉及模块：`game_auto_start.ahk`、`config.ahk`、`settings/saver.ahk`、`gui.ahk`、`main.ahk`

- [x] **前置**：AFA 以管理员权限运行，已填写有效的 `Arknights.exe` 路径，AFA 当前已退出。
- [x] **操作**：勾选新选项，确认 Windows 审核提示并保存；关闭 AFA。
- [x] **预期**：设置保存成功，Windows“进程创建成功审核”启用，计划任务 `ArknightsFrameAssistant-AutoStartWithGame-<当前用户SID>` 创建成功。
- [x] **操作**：从原有游戏入口启动明日方舟。
- [x] **预期**：AFA 自动启动且只出现一个实例，不弹出设置窗口、更新公告或自动更新提示，托盘图标正常显示。
- [x] **操作**：关闭游戏。
- [x] **预期**：在“随游戏进程关闭自动退出”开启时，AFA 自动退出。
- [x] **操作**：再次启动游戏。
- [x] **预期**：AFA 可以再次被自动启动。

##### 异常路径

- [x] **异常**：启动其他程序或不同路径下的同名程序 → **预期**：AFA 不启动。
- [x] **异常**：计划任务服务不可用或 Windows 审核设置失败 → **预期**：新选项不保存，并显示明确错误。
- [x] **异常**：AFA 已经运行时启动游戏 → **预期**：不产生第二个 AFA 实例。
- [x] **异常**：关闭功能后仍有一次遗留事件触发 → **预期**：AFA 清理计划任务后立即退出，不显示托盘或设置窗口。
- [x] **异常**：游戏已运行时启动 AFA，且两个自动启动选项都开启 → **预期**：不重复启动游戏或 AFA。

### 流程：关闭和校准自动启动

> 涉及模块：`game_auto_start.ahk`、`config.ahk`、`settings/saver.ahk`

- [x] **前置**：新功能已启用，计划任务存在。
- [x] **操作**：取消勾选新选项并保存。
- [x] **预期**：计划任务被删除，后续启动游戏不会自动启动 AFA；进程创建成功审核保持开启。
- [x] **操作**：开启新功能后修改游戏路径，再次保存。
- [x] **预期**：计划任务事件过滤器更新为新的游戏路径。
- [x] **操作**：启用新功能后移动 AFA EXE，手动启动移动后的 AFA。
- [x] **预期**：任务动作路径被校准为新的 EXE 路径。
