# 测试清单：GitHub Token 安全存储

> 对应 Issue #166：使用 Windows DPAPI 保护 GitHub Token，旧版明文配置自动迁移。
> 关联 Issue #181：设置保存过程中配置项被清空的问题。

| 项目 | 信息 |
|------|------|
| AFA 版本 | `v1.5.9` |
| AutoHotkey 版本 | `2.0.26` |
| Windows 版本 | `Windows 11 Pro for Workstations 24H2（构建 26100.4351）` |

## 单元与配置测试

### DPAPI 加密与解密

- [x] **操作**：在设置中输入有效 GitHub Token，保存设置并退出 AFA。
- [x] **预期**：`%AppData%\ArknightsFrameAssistant\PC\Settings.ini` 中不存在 `GitHubToken=` 明文键，存在 `GitHubTokenProtected=dpapi:v1:` 加密键。
- [x] **操作**：重新启动 AFA，打开更新设置。
- [x] **预期**：Token 可以正常加载，密码框仍显示掩码，GitHub Token 验证和更新检查正常。
- [ ] **操作**：保存其他设置多次，并触发自动更新中的忽略版本保存。
- [ ] **预期**：Token 不会被重复加密，始终可以正常解密使用。

### 空值和替换

- [x] **操作**：清空 Token 并保存设置。
- [x] **预期**：Token 相关配置被清理，更新检查仍可使用无 Token 模式，不产生错误提示。
- [x] **操作**：输入新的有效 Token 并保存。
- [x] **预期**：旧密文被替换，新 Token 可以正常验证和使用。

## 迁移测试

### 旧版明文 Token 自动迁移

- [x] **前置**：备份设置文件，并在 `[Main]` 中准备旧版 `GitHubToken=<测试 Token>`，移除 `GitHubTokenProtected`。
- [x] **操作**：启动新版 AFA。
- [x] **预期**：启动完成后生成 `GitHubTokenProtected=dpapi:v1:`，旧 `GitHubToken` 明文键被删除。
- [x] **操作**：重启 AFA 并验证 GitHub Token。
- [x] **预期**：迁移后的 Token 可正常使用，设置界面不显示迁移错误。

### 迁移失败保护

- [x] **操作**：使用权限不足或不可写的配置文件目录启动 AFA，配置中保留旧版明文 Token。
- [x] **预期**：旧明文不会被删除，AFA 显示安全迁移失败提示，并在后续启动继续重试迁移。
- [x] **操作**：恢复配置文件写权限并重启 AFA。
- [x] **预期**：迁移成功，明文键被删除。
- [x] **操作**：在只读配置状态下关闭更新公告，或勾选“直到下次更新前不再弹出”。
- [x] **预期**：公告窗口可以正常关闭，不出现未处理的 `Error (5) 拒绝访问`；若勾选项无法保存，仅提示配置未保存。
- [ ] **操作**：在只读配置状态下触发“忽略此版本”。
- [ ] **预期**：更新提示可以关闭，忽略状态保存失败时显示明确警告，不阻塞 AFA 主程序。

## 异常路径

### 密文损坏或跨用户读取

- [x] **操作**：修改 `GitHubTokenProtected` 的 Base64 内容，或将配置复制到其他 Windows 用户下启动 AFA。
- [x] **预期**：AFA 不崩溃，不把密文发送给 GitHub，Token 显示为不可用并提示重新输入。
- [x] **操作**：重新输入有效 Token 并保存。
- [x] **预期**：错误状态消失，新 Token 可以正常使用。

### 解密失败时保存保护

- [x] **前置**：使 `GitHubTokenProtected` 损坏或在其他 Windows 用户下启动 AFA，确保 Token 无法解密。
- [x] **操作**：只修改其他设置，不重新输入 GitHub Token，点击“应用设置”或“保存并关闭”。
- [x] **预期**：保存被阻止，原有 `GitHubTokenProtected` 内容保持不变，并提示先重新输入 Token。
- [x] **操作**：重新输入有效 Token 并保存设置。
- [x] **预期**：新 Token 加密保存，后续更新检查可以正常使用。
- [x] **操作**：在 Token 无法解密时，同时修改“随明日方舟自动启动小助手”设置并尝试保存。
- [x] **预期**：DPAPI 预检在计划任务变更前失败，计划任务和原有 `GitHubTokenProtected` 均保持不变。

### 配置保存原子性（Issue #181）

- [x] **前置**：备份 `Settings.ini`，并确保 `DismissedChangelogVersion` 已保存为非空值。
- [ ] **操作**：在自动更新和游戏状态监控运行期间多次保存其他设置。
- [ ] **预期**：`DismissedChangelogVersion` 不会被清空，`[Main]` 中的其他配置项也不会丢失。
- [x] **操作**：模拟保存过程中的写入失败或中断。
- [x] **预期**：原 `Settings.ini` 保持不变，不会留下空的或部分写入的 `[Main]`。

- [x] **操作**：关闭测试专用构建，恢复正常 DPAPI 调用并重新启动 AFA。
- [x] **预期**：输入或保存 Token 可以成功完成。
