## 问题反馈

### 问题1：点击 Edit 控件触发按键绑定逻辑，导致主窗口异常置顶
- 测试项：更新提示弹窗 Edit 控件显示 changelog
- 预期：点击 Edit 控件仅选中/滚动文本，不影响其他窗口
- 实际：点击 Edit 时触发 `key_bind.ahk` 的全局 `WM_LBUTTONDOWN` 消息处理，InputHook 被激活，主设置窗口被拉到前台，且 Edit 内容被替换为"请按键"
- 根因：`OnMessage(0x0201, WM_LBUTTONDOWN)` 对进程中所有 GUI 的 Edit 控件生效，未过滤非设置窗口
- 修复：在 `WM_LBUTTONDOWN` 中增加 `KeyBinder.ControlObj.Gui.Hwnd != GuiManager.MainGui.Hwnd` 检查，仅处理设置窗口内的 Edit 控件
- [x] 已修复

### 问题2：更新内容未显示版本号
- 测试项：更新提示弹窗 Edit 控件内容
- 预期：每条更新内容前应显示对应版本号（如 `## v1.4.7`）
- 实际：仅显示 release body 内容，无版本号标识
- 修复：`_BuildChangelogBody` 和 `_ReadAndBuildBody` 中拼接 body 时添加 `## tag_name` 标题行
- [x] 已修复

### 问题3：按钮与 Edit 控件边缘未对齐，间距过近
- 测试项：更新提示弹窗按钮布局
- 预期：按钮左右边缘与 Edit 控件对齐，与 Edit 底部有合适间距
- 实际：按钮居中排列，与 Edit 左右边不对齐，且紧贴 Edit 底部
- 修复：changelogBody 非空时 `startX=20`，`btnGap=30`，按钮跨度 20~380 与 Edit 对齐，`btnY=300` 增加与 Edit 间距
- [x] 已修复
