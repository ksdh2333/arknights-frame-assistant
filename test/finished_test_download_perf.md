## 问题反馈

### 问题1：WinHttpRequest 兼容性导致下载超时
- 测试项：单元测试 — 下载进度条显示
- 预期：进度条逐步增长，下载正常完成
- 实际：进度条卡在 0%，重试3次后提示超时
- 原因：`WinHttp.WinHttpRequest.5.1` 在 Windows 11 25H2 下兼容性问题
- 修复：切回 `MSXML2.ServerXMLHTTP.6.0` COM 对象
- [x] 已解决

### 问题2：关闭下载窗口不取消后台下载（已有问题）
- 测试项：异常路径 — 下载过程中关闭下载窗口
- 预期：下载应被取消
- 实际：窗口关闭后下载仍在后台继续，超时重试时下载窗口又自动弹出
- 修复：为 DownloadingDialog 注册 OnEvent("Close") → 触发取消流程 (`updater_ui.ahk:181`)
- [x] 已解决

### 问题3：点击"手动下载"会关闭窗口但不取消下载
- 测试项：下载窗口交互
- 预期：点击"手动下载"应打开浏览器下载，不关闭当前下载进度窗口，不取消正在进行的下载
- 实际：点击"手动下载"后窗口关闭，但后台下载仍在继续
- 修复：删除 OnManualDownload() 中的 CloseDownloadingDialog() 调用 (`updater_ui.ahk:188-191`)
- [x] 已解决

### 问题4：下载请求偶发连接超时（复现）
- 测试项：单元测试 — 下载进度条显示
- 预期：下载正常完成
- 实际：测试后期再次出现连接超时（可能因频繁测试触发 GitHub 速率限制，待确认）
- 修复：ConnStallTimeout 从 10s 提高到 20s，适应国内访问 GitHub 的网络条件 (`downloader.ahk:41`)
- [ ] 疑似网络波动，跳过
